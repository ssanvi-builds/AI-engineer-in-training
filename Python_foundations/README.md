## Project instructions:

This is the way I approach every project in this folder:

- Read through the ATBSWP chapter and stop when you reach a project output. Make a screenshot of it and save it as chapterNprojectgoal.png
- Take the chapterNprojectgoal.png and try to replicate the output with the current knowledge you have.
- If you get errors, try to debug them yourself. Look up the documentation of the functions, methods and tools you're using.
- If you're stuck, ask your preferred LLM to guide you through the problem *without* giving an answer. The goal is to think, understand and learn.
- Document the problems and insights you get in the notes.md.
- Once you have a working project, check out the solution proposed in the book and look for differences with your code focusing on:
    - Code efficiency. Less is more.
    - Code clarity. You should be able to understand it while reading it.
- If required, write a revised version of your code with all the improvements.
- Commit it to your personal repo.
